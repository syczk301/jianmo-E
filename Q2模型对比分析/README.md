# 轴承故障诊断模型对比分析

本文件夹包含了五种不同机器学习模型的完整实现和对比分析，用于轴承故障诊断任务。

## 📁 文件结构

```
模型对比分析/
├── 决策树模型.py          # 决策树分类器实现
├── 随机森林模型.py        # 随机森林分类器实现  
├── AdaBoost模型.py        # AdaBoost分类器实现
├── XGBoost模型.py         # XGBoost分类器实现
├── GBDT模型.py           # 梯度提升树分类器实现
├── 模型综合对比.py        # 所有模型的综合对比分析
└── README.md             # 本说明文件
```

## 🤖 模型介绍

### 1. 决策树 (Decision Tree)
- **特点**: 简单直观，易于理解和解释
- **优势**: 不需要数据预处理，能处理数值和分类特征
- **适用场景**: 需要模型可解释性的场合

### 2. 随机森林 (Random Forest)  
- **特点**: 集成多个决策树，通过投票决定最终结果
- **优势**: 抗过拟合能力强，特征重要性评估
- **适用场景**: 数据量较大，需要稳定性能的场合

### 3. AdaBoost (Adaptive Boosting)
- **特点**: 自适应提升算法，逐步纠正前一个分类器的错误
- **优势**: 能将弱分类器提升为强分类器
- **适用场景**: 二分类问题，对噪声敏感度较高

### 4. XGBoost (Extreme Gradient Boosting)
- **特点**: 优化的梯度提升框架，性能卓越
- **优势**: 处理缺失值，防过拟合，速度快
- **适用场景**: 竞赛和实际应用中的首选算法

### 5. GBDT (Gradient Boosting Decision Tree)
- **特点**: 梯度提升决策树，逐步优化损失函数
- **优势**: 预测精度高，能处理各种数据类型
- **适用场景**: 回归和分类任务，对精度要求较高

## 🚀 使用方法

### 环境要求
```bash
pip install numpy pandas scikit-learn matplotlib seaborn
pip install xgboost  # 可选，用于XGBoost模型
```

### 运行单个模型
```bash
python 决策树模型.py
python 随机森林模型.py
python AdaBoost模型.py
python XGBoost模型.py
python GBDT模型.py
```

### 运行综合对比
```bash
python 模型综合对比.py
```

## 📈 数据要求

模型期望的数据格式：
- **输入**: Excel文件 `../data/merged_data.xlsx`
- **特征**: 数值型特征矩阵
- **标签**: 分类标签列 `label`

数据应包含轴承故障的多种类别（如正常、内圈故障、外圈故障、滚动体故障等）。

## ⚙️ 参数调优

每个模型都包含网格搜索参数优化：

- **决策树**: 最大深度、最小分裂样本数、分裂准则等
- **随机森林**: 树的数量、最大深度、特征采样比例等  
- **AdaBoost**: 基学习器数量、学习率、算法类型等
- **XGBoost**: 树数量、深度、学习率、正则化参数等
- **GBDT**: 提升轮数、学习率、子采样比例等

## 📋 结果解读

### 性能指标说明
- **准确率**: 正确预测的比例
- **精确率**: 预测为正的样本中实际为正的比例
- **召回率**: 实际为正的样本中被正确预测的比例  
- **F1分数**: 精确率和召回率的调和平均数

### 选择建议
1. **追求准确率**: 选择F1分数最高的模型
2. **需要可解释性**: 选择决策树
3. **数据量大**: 选择随机森林或XGBoost
4. **计算资源有限**: 选择决策树或随机森林
5. **追求极致性能**: 选择XGBoost或GBDT

## 🔧 自定义修改

### 添加新模型
1. 创建新的Python文件
2. 按照现有模板实现训练和评估函数
3. 在 `模型综合对比.py` 中添加新模型

### 修改评估指标
在各模型文件的 `evaluate_model` 函数中添加新的评估指标。

### 调整可视化
修改各绘图函数的参数，如颜色、图表大小、字体等。

## 📞 注意事项

1. 确保数据文件路径正确
2. 某些模型（如XGBoost）需要额外安装
3. 网格搜索可能耗时较长，可适当减少参数范围
4. 大数据集建议使用采样或分布式计算
5. 结果图片会保存在运行目录下
